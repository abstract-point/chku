# ЧКУ — бизнес-правила клуба

ЧКУ — «Читальный клуб умничек» — private reading club platform. Клуб читает одну книгу за цикл, отмечает прогресс, проводит встречу, ставит оценки и сохраняет историю в архиве.

Главная особенность проекта: книги выбираются **по очереди**, а не голосованием. Голосование допустимо для организационных вопросов встречи, но не для выбора книги.

Главное правило клуба:

> Следующая книга может быть утверждена только если никто из активных участников клуба раньше её не читал.

---

## Жизненный цикл чтения

Циклы идут друг за другом. В нормальном состоянии у клуба есть не больше одного незавершённого цикла.

Статусы цикла:

- `proposed` — книга предложена и проходит проверку «никто не читал»;
- `active` — книга утверждена, клуб читает её и отмечает прогресс;
- `completed` — встреча завершена, оценки собраны, цикл попал в архив.

### 1. Выбор книги (`proposed`)

- Когда предыдущий цикл завершён, система берёт первого активного участника из очереди выбора.
- Для этого участника берётся первая книга из его личной очереди со статусом `queued`.
- Создаётся новый цикл со статусом `proposed` и кандидат на проверку этой книги.
- Кандидат сразу отправляется активным участникам клуба.
- Автор предложения автоматически считается ответившим `not_read`.
- Активные участники отвечают:
  - `not_read` — не читал;
  - `read` — читал;
  - `pending` — ещё не ответил.
- Если хотя бы один активный участник ответил `read`, кандидат отклоняется, его позиция в личной очереди получает статус `rejected`, а система пробует следующую книгу того же выбирающего.
- Если есть `pending`, книгу нельзя утвердить.
- Если все активные участники ответили `not_read`, кандидат переходит в ожидание подтверждения автора предложения.
- После подтверждения автора цикл становится `active`, а позиция в личной очереди получает статус `approved`.

Если у участника, чья очередь подошла, нет книг в личной очереди, новый цикл не создаётся. Интерфейс должен показать этому участнику действие для пополнения очереди.

### 2. Активное чтение (`active`)

- В активном цикле участники читают утверждённую книгу.
- Для каждого активного участника создаётся запись прогресса.
- Прогресс можно хранить как процент, текущую страницу и короткую заметку.
- Статусы прогресса:
  - `not_started`;
  - `reading`;
  - `finished`;
  - `abandoned`.
- Dashboard показывает прогресс клуба как лидерборд.
- Первые три места в лидерборде должны быть помечены ненавязчиво, но явно: золото, серебро, бронза. Визуально можно обыгрывать логотип с совой из `apps/chku-frontend/public/`.

### 3. Встреча и завершение (`meeting -> completed`)

- Встречу создаёт администратор на этапе `active`.
- Для одного активного цикла может быть только одна текущая встреча.
- Администратор может редактировать встречу до завершения.
- Интерфейс встречи должен поддерживать два явных действия администратора:
  - начать встречу;
  - закончить встречу.
- Встречу можно начать только если минимум 2 участника отметились RSVP `attending`.
- Завершение встречи завершает цикл и переводит его в `completed`.
- Завершить встречу можно только если минимум 2 участника встречи отметились RSVP `attending`.
- Завершить встречу можно только когда все участники встречи поставили оценку.
- Участниками встречи считаются RSVP `attending`.
- Оценка обязательна и находится в диапазоне от 1 до 10.
- Текст отзыва необязателен. Если участник поставил оценку без текста, система сохраняет отзыв `Промолчал`.
- После завершения встречи книга попадает в архив, а система запускает следующий `proposed` цикл по очереди выбора.

---

## Очередь выбора и личные очереди книг

- В очереди выбора участвуют только активные участники.
- Новый участник обычно добавляется в конец очереди.
- Неактивного участника можно пропустить.
- История выборов, оценок, отзывов и встреч сохраняется, даже если участник покинул клуб.
- У каждого участника есть личная очередь книг.
- Когда очередь выбора доходит до участника, система берёт первую доступную книгу из его личной очереди.

---

## Архив и рейтинги

- Завершённый цикл остаётся в архиве клуба.
- Рейтинг книги считается по оценкам участников в рамках конкретного цикла.
- Один участник может поставить одну оценку на книгу в рамках цикла.
- Один участник может иметь один отзыв на книгу в рамках цикла.
- Исторические данные не удаляются при деактивации участника.

## Продуктовый фокус MVP

Главный сценарий:

1. Завершается встреча по текущей книге.
2. Завершается текущий цикл.
3. Система берёт первую книгу из очереди следующего выбирающего.
4. Начинается новый цикл `proposed`.
5. Активные участники отвечают, читали ли книгу.
6. Если никто не читал, автор предложения подтверждает книгу.
7. Цикл становится `active`.
8. Участники отмечают прогресс, а Dashboard показывает лидерборд.
9. Администратор создаёт, редактирует, начинает и завершает встречу.
10. Участники встречи ставят оценки.
11. Встреча и цикл завершаются, книга попадает в архив.

## UI/UX принципы

Интерфейс должен быть современным, минималистичным, спокойным, стильным, читаемым и без визуального шума.

Не делать интерфейс похожим на корпоративную CRM, маркетплейс книг, социальную сеть, тяжёлую админку, детский книжный сайт или винтажный сайт со свитками.

Dashboard должен быстро отвечать:

- что клуб читает сейчас;
- какой мой прогресс;
- кто лидирует по прогрессу чтения;
- когда следующая встреча;
- кто выбирает следующую книгу;
- прошла ли книга проверку по правилу «никто не читал».
