# ЧКУ — краткое описание проекта

ЧКУ — «Читальный клуб умничек» — веб-сервис для частного читального клуба.

Участники клуба читают одну книгу за цикл, отмечают прогресс, встречаются для обсуждения, выставляют оценки и оставляют отзывы.

Главная особенность проекта: книги выбираются **по очереди**, а не голосованием. Каждый участник в свой ход предлагает следующую книгу.

Главное правило клуба:

> Следующая книга может быть утверждена только если никто из активных участников клуба раньше её не читал.

---

## Основной функционал

MVP должен включать:

- авторизацию пользователей;
- список участников клуба;
- очередь выбора книг;
- добавление книги-кандидата;
- проверку книги по правилу «никто не читал»;
- текущую книгу клуба;
- прогресс чтения участников;
- назначение встречи;
- оценки после прочтения;
- короткие отзывы;
- архив прочитанных книг;
- рейтинг книг клуба.

---

## Основные сущности

- `User` — пользователь системы.
- `Club` — читальный клуб.
- `ClubMember` — участник клуба.
- `Book` — книга.
- `BookCandidate` — предложенная следующая книга.
- `BookCandidateResponse` — ответ участника: читал / не читал / не уверен.
- `TurnOrderItem` — элемент очереди выбора.
- `ReadingCycle` — цикл чтения одной книги.
- `ReadingProgress` — прогресс участника по текущей книге.
- `Meeting` — встреча клуба.
- `Rating` — оценка книги участником.
- `Review` — отзыв участника.

## Главные бизнес-правила

### Выбор книги

- Книги выбираются участниками по очереди.
- Голосование за книгу не является основной механикой.
- Текущий выбирающий предлагает книгу-кандидата.
- Книга-кандидат отправляется на проверку участникам клуба.
- Каждый активный участник должен ответить:
  - `not_read` — не читал;
  - `read` — читал;
  - `not_sure` — не уверен;
  - `pending` — ещё не ответил.

### Утверждение книги

- Если хотя бы один активный участник ответил `read`, книгу нельзя утвердить.
- Если есть ответы `pending` или `not_sure`, книгу нельзя утверждать автоматически.
- Если все активные участники ответили `not_read`, книгу можно утвердить.
- После утверждения книга становится текущей или следующей книгой клуба.

### Очередь выбора

- В очереди участвуют только активные участники.
- Новый участник обычно добавляется в конец очереди.
- Неактивного участника можно пропустить.
- История выбора книг должна сохраняться.
- Если участник вышел из клуба, его прошлые выборы и отзывы не удаляются.

### Прогресс чтения

Статусы прогресса:

- `not_started`
- `reading`
- `finished`
- `abandoned`

Прогресс можно хранить как:

- процент;
- текущую страницу;
- главу;
- короткую заметку.

Для MVP достаточно процента или страниц.

### Встречи

Встреча привязана к книге или циклу чтения.

Встреча содержит:

- дату;
- время;
- место или ссылку;
- список участников;
- RSVP-статусы;
- вопросы для обсуждения;
- заметки после встречи.

Голосование допустимо для выбора даты/времени встречи, но не является основной механикой выбора книги.

### Оценки и отзывы

- Один участник ставит одну оценку на одну книгу в рамках цикла чтения.
- Оценка может быть от 1 до 10.
- Отзыв короткий и текстовый.
- Средний рейтинг книги считается по оценкам участников.
- После завершения цикла книга попадает в архив клуба.

## Продуктовый фокус

Главный сценарий MVP:

1. Участник видит, что сейчас его очередь выбрать книгу.
2. Он предлагает книгу.
3. Остальные участники подтверждают, что не читали её.
4. Если никто не читал, книга утверждается.
5. Клуб читает книгу.
6. Участники отмечают прогресс.
7. Клуб проводит встречу.
8. Участники ставят оценки и пишут отзывы.
9. Книга попадает в архив и рейтинг клуба.

## UI/UX принципы

Интерфейс должен быть:

- современным;
- минималистичным;
- спокойным;
- стильным;
- читаемым;
- без визуального шума.

Не делать интерфейс похожим на:

- корпоративную CRM;
- маркетплейс книг;
- социальную сеть;
- тяжёлую админку;
- детский книжный сайт;
- винтажный сайт со свитками, перьями и старой бумагой.

Dashboard должен быстро отвечать на вопросы:

- что мы сейчас читаем;
- какой мой прогресс;
- когда следующая встреча;
- кто выбирает следующую книгу;
- прошла ли книга проверку по правилу «никто не читал».
